# Session Continuity

Claude Code sessions lose context in three ways: compaction (1M token limit reached), graceful exit, and crashes. The session continuity system uses three hooks to write structured handoff documents before context is lost, and reads them back on resume.

> **Status (April 2026):** The triple-handoff system is currently disabled in production and under re-assessment. While Opus 4.6 advertises a 1M context window, model reliability degrades significantly past 20-30% of that window--making compaction more frequent in practice than the raw number suggests. The system's cost (OpenRouter API calls per Stop event) and firing frequency are being re-evaluated. The hooks and infrastructure remain in the repo for teams that want them. If you enable them, expect ~$0.01-0.03 per handoff in OpenRouter costs.

## The Triple-Handoff System

| Event | Hook | Trigger |
|-------|------|---------|
| `PreCompact` | `precompact-handoff.py` | Context window about to be compacted |
| `SessionEnd` | `precompact-handoff.py` | Graceful exit (user quits or `/exit`) |
| `Stop` | `stop-handoff.py` | Every model response, 3-min cooldown between writes (10-min idle gate) |

All three write to the same location: `~/.claude/handoffs/{session_id}.yaml`. Each write overwrites the previous--only the latest state matters.

`stop-handoff.py` is the safety net. It fires after every model response with a 3-minute cooldown (plus a 10-minute idle gate). If the session crashes between compactions, the most recent Stop handoff captures the state.

## Handoff YAML Schema

```yaml
session_id: "abc123def"
timestamp: "2026-04-06T14:00:00"
trigger: "stop"           # "compact", "session_end", or "stop"
project: "worldwarwatcher"
handoff_count: 7           # increments each write -- vitality marker

objective: What was being worked on
completed:
  - Task 1 that was finished
  - Task 2 that was finished
decisions:
  - Key decision and reasoning
blockers:
  - What's stuck or unclear
next_steps:
  - Immediate next action
  - Follow-up after that
```

The YAML is generated by calling an LLM via OpenRouter to summarize the session transcript. This runs outside the Claude Code model so it doesn't consume context tokens.

## INDEX.md

`~/.claude/handoffs/INDEX.md` is an auto-maintained table of all sessions. Most recent first.

```markdown
| Date | Project | Session | Trigger | Directory | Summary |
|------|---------|---------|---------|-----------|---------|
| 2026-04-06T14:00 | worldwarwatcher | `abc123de` | stop | .../worldwarwatcher | Implementing phosphor vigil... |
| 2026-04-06T12:30 | claudicle | `def456gh` | compact | .../claudicle | Building test suite... |
```

New sessions check INDEX.md to find the most recent handoff for their project directory and resume from it.

## Resuming from a Handoff

On `SessionStart` with a `resume` trigger, the system:
1. Reads `~/.claude/handoffs/INDEX.md`
2. Finds the handoff matching the current session ID (or most recent for the directory)
3. Reads the YAML file
4. Injects the handoff content into the system prompt via `additionalContext`

Claude then has: objective, what was completed, key decisions, blockers, and next steps. It resumes work without the user re-explaining context.

## Git Tracking System

Three hooks track git operations across all directories, building a bidirectional session-to-repo index:

```
PreToolUse:Bash  → git-track.sh       → ~/.claude/git-tracking.jsonl (append)
PostToolUse:Bash → git-track-post.sh  → Capture commit hashes post-execution
SessionEnd       → git-track-rebuild.py → ~/.claude/git-tracking-index.json (rebuild)
```

The JSONL log records every git command: repo path, branch, operation type, commit hashes, session ID. The index provides lookup in both directions: which sessions touched a repo, which repos a session touched.

Query the tracking data with `/session-report` for a markdown dashboard of session activity, git operations, and commit history.

## Plan Auto-Naming

Two hooks manage plan file naming:

**`plan-rename.py`** (PostToolUse: Write/Edit) -- When Claude Code creates a plan file, it gets a random slug like `zesty-kindling-giraffe.md`. This hook extracts the H1 header and renames it to a dated slug: `2026-04-06-implement-phosphor-vigil.md`. A forwarding symlink preserves the old name for mid-session references.

**`plan-session-rename.py`** (PostToolUse: Write) -- When a plan is created, this hook extracts the H1 header and sets the session's `customTitle` in `sessions-index.json`. The session tracker then shows meaningful names instead of random strings.

**`plan-cleanup-symlinks.py`** (SessionEnd) -- Removes forwarding symlinks left by plan-rename.py after the session ends.

## Setup

Register the hooks in `~/.claude/settings.json`:

```json
{
  "hooks": {
    "PreCompact": [
      {"type": "command", "command": "python3 ~/.claude/hooks/precompact-handoff.py", "async": false}
    ],
    "SessionEnd": [
      {"type": "command", "command": "python3 ~/.claude/hooks/precompact-handoff.py", "async": false}
    ],
    "Stop": [
      {"type": "command", "command": "python3 ~/.claude/hooks/stop-handoff.py", "async": true}
    ]
  }
}
```

The handoff hooks require an OpenRouter API key (set `OPENROUTER_API_KEY` in your environment) for transcript summarization. Without it, handoffs will fail silently.

See `hooks/README.md` for the full hook inventory and wiring details.
